1. `phone:ping` handler:
   - Listens for `socket.on('phone:ping', ({ roomId }) => { ... })`
   - Broadcasts to the room EXCEPT the sender: `io.to(roomId).except(socket.id).emit('server:ping', {})`
     (This notifies the Totem that a ping arrived so it can flash/beep)
   - Sends pong back to the SENDER only: `socket.emit('server:pong', {})`
     (Phone uses this to calculate RTT and re-enable button)
   - Add console.log: `console.log(\`Ping from phone in room: ${roomId}\`)`

2. `phone:latency` handler:
   - Listens for `socket.on('phone:latency', ({ roomId, latencyMs }) => { ... })`
   - Broadcasts to room EXCEPT sender: `io.to(roomId).except(socket.id).emit('server:latency', { latencyMs })`
     (Totem receives the latency value to display)
   - Add console.log: `console.log(\`Latency reported: ${latencyMs}ms in room ${roomId}\`)`

Event names follow the established `noun:verb` convention:
- `phone:ping` (phone -> server) — phone initiated the ping
- `server:pong` (server -> phone) — server replies to phone
- `server:ping` (server -> totem) — server notifies totem of incoming ping
- `phone:latency` (phone -> server) — phone reports calculated RTT
- `server:latency` (server -> totem) — server forwards RTT to totem
