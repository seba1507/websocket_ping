## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#1a1a1a` | Body background — both Phone and Totem |
| Secondary (30%) | `#333` / `#555` / `#888` | Borders, muted text, idle status text |
| Accent (10%) | `#39ff14` | Accent reserved-for list below |
| Destructive | `#ff4444` | Error state on `#status` only |

Accent `#39ff14` is reserved for:
1. `#ping-btn` background (idle state)
2. `#latency-display` text color on Totem
3. `#status.connected` text color on both Phone and Totem
4. Totem full-screen flash: body background override for 200ms
5. Phone full-screen flash: body background override for 200ms (on pong arrival)

Accent `#2bcc0e` (dark neon green) reserved for:
- `#ping-btn:active` background only (pressed state)

Text on accent: `#111` (near-black) — used on `#ping-btn` label only.
Default text: `#f0f0f0` on dark background.

Flash behavior (no CSS transitions/animations — instant swap only):
- Totem: `document.body.style.background = '#39ff14'` → restore to `#1a1a1a` after 200ms
- Phone: same pattern on `server:pong` receipt, concurrently with haptic

## Interaction Contract

### Phone — `#ping-btn`

| State | Background | Opacity | Cursor | Text |
|-------|-----------|---------|--------|------|
| Hidden (pre-join) | — | 0 (display: none) | — | SEND PING |
| Idle (ready) | `#39ff14` | 1 | pointer | SEND PING |
| Pressed (:active) | `#2bcc0e` | 1 | pointer | SEND PING |
| Awaiting pong (disabled) | `#39ff14` | 0.4 | default (no change) | SEND PING |

Disabled state: `button.disabled = true` + `style.opacity = '0.4'` — set immediately after tap.
Re-enable: `button.disabled = false` + `style.opacity = '1'` — set when `server:pong` arrives (not on timeout).

No CSS transition on the opacity change — instant swap only (consistent with no-animation rule).

Source: CONTEXT.md `<decisions>` — "Button state during flight".

### Phone — Pong Feedback Sequence

On `server:pong` arrival:
1. Calculate RTT: `Date.now() - pingTimestamp`
2. Set `#pong-feedback` text to RTT value (e.g. `42 ms`)
3. Flash body background: `#1a1a1a` → `#39ff14`, restore after 200ms (instant swap, no transition)
4. Vibrate: `navigator.vibrate(80)` — silent fail if unsupported
5. Re-enable `#ping-btn`

`#pong-feedback` text color: `#39ff14` (neon green) while showing RTT result. Persists until next ping.

### Totem — Flash + Latency Update Sequence

On `phone:ping` routed from server:
1. Flash body background: `#1a1a1a` → `#39ff14`, restore after 200ms (instant swap, no transition)
2. Play Web Audio API beep: 880Hz sine wave, 80ms, exponential fade-out envelope
3. On `phone:latency` routed from server: set `#latency-display` textContent to `⏱️ {N} ms`

Latency display persists — shows last measurement until overwritten. Never resets to blank between pings.

Source: CONTEXT.md `<decisions>`, `<specifics>`, REQUIREMENTS.md INT-02 through INT-05.
